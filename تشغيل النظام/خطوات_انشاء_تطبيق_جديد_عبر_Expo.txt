========================================================================
خطوات إنشاء تطبيق جديد عبر Expo — انطلاقاً من نفس نظام SmartPDF
========================================================================
آخر تحديث للمعلومات: يوليو 2026 (تم التحقق من خطوات Expo/EAS الحالية)


القسم 1: إنشاء حساب على Expo
------------------------------------------------------------------------
1. افتح المتصفح واذهب إلى: https://expo.dev
2. اضغط على زر "Sign Up" (تسجيل) أعلى يمين الصفحة.
3. سجّل باستخدام:
   - بريد إلكتروني + كلمة مرور، أو
   - حساب GitHub (يُفضّل هذا الخيار لأنه يسهّل ربط المشروع لاحقاً
     بـ GitHub Actions إن احتجت أتمتة النشر مستقبلاً).
4. أكّد بريدك الإلكتروني من الرسالة التي تصلك.
5. بعد الدخول، ستصل للوحة تحكم expo.dev — هذا المكان اللي رح تراقب فيه
   كل عمليات البناء (Builds) لاحقاً، وتدير فيه حسابك ومشاريعك.

ملاحظة: الحساب نفسه مجاني بالكامل. الدفع بيصير بس لو احتجت:
   - عضوية Google Play Developer (رسوم لمرة واحدة: 25$)
   - عضوية Apple Developer Program (رسوم سنوية: 99$) — مطلوبة فقط
     لو بدك تنشر على App Store أو تختبر على آيفون حقيقي عبر EAS.


القسم 2: تجهيز جهاز الكمبيوتر
------------------------------------------------------------------------
1. ثبّت Node.js (الإصدار LTS الحالي) من: https://nodejs.org
   تحقق من التثبيت بكتابة في الطرفية (Terminal / CMD):
       node -v
       npm -v

2. ثبّت Git من: https://git-scm.com (لازم لرفع الكود لاحقاً على GitHub
   واستخدامه مع Claude Code إن رغبت).

3. (اختياري لكن مفيد) ثبّت VS Code كمحرر أكواد:
   https://code.visualstudio.com

4. ثبّت تطبيق "Expo Go" على جوالك (أندرويد أو آيفون) من متجر التطبيقات
   — يخليك تجرّب التطبيق فوراً أثناء التطوير بدون أي بناء (Build).


القسم 3: تثبيت أدوات Expo على جهازك
------------------------------------------------------------------------
1. افتح الطرفية (Terminal) وثبّت EAS CLI (أداة البناء والنشر السحابية):
       npm install -g eas-cli

   (بديل بدون تثبيت عام: استخدم npx eas-cli@latest في كل أمر بدل eas)

2. سجّل دخولك من نفس الطرفية بنفس بيانات حساب expo.dev اللي أنشأته:
       eas login

3. تأكد إنك مسجّل دخول فعلاً:
       eas whoami


القسم 4: إنشاء مشروع Expo جديد
------------------------------------------------------------------------
1. من المجلد اللي بدك تحط فيه المشروع، نفّذ:
       npx create-expo-app MyNewApp

   (استبدل MyNewApp باسم تطبيقك الفعلي، بدون مسافات أو رموز عربية
   في اسم المجلد نفسه — الاسم المعروض للمستخدم بالتطبيق شي تاني تماماً
   وممكن يكون عربي، هيدا بس اسم المجلد التقني).

2. ادخل لمجلد المشروع:
       cd MyNewApp

3. شغّل خادم التطوير للتأكد إن كل شي شغّال:
       npx expo start

4. افتح تطبيق Expo Go على جوالك وامسح رمز QR الظاهر بالطرفية —
   رح يفتح تطبيقك فوراً على جوالك الحقيقي بدون أي بناء أو تثبيت.


القسم 5: تجهيز هيكلة المشروع (بنفس نمط نظام SmartPDF)
------------------------------------------------------------------------
نظام SmartPDF مبني على:
   - Expo Router (نظام التوجيه المعتمد على الملفات) — يجي افتراضياً
     مع create-expo-app الحديث.
   - مجلد lib/ يحوي كل المنطق المشترك:
       lib/i18n.tsx      -> نظام الترجمة (عربي/إنجليزي)
       lib/theme.tsx     -> نظام الثيمات (الألوان)
       lib/trial.tsx     -> نظام التجربة المجانية (مدة + تحقق من الخادم)
       lib/config.ts     -> رابط الخادم (SERVER_URL) وإعدادات عامة
   - مجلد app/(tabs)/ يحوي شاشات التبويبات الرئيسية:
       index.tsx     -> الشاشة الرئيسية (قائمة الأدوات)
       settings.tsx  -> شاشة الإعدادات
       files.tsx     -> شاشة الملفات المحفوظة (إن وجدت)

خطوات عملية لتجهيز نفس الهيكلة بمشروعك الجديد:
1. أنشئ مجلد lib/ داخل المشروع.
2. ثبّت الحزم الأساسية اللي يعتمد عليها نظام SmartPDF:
       npx expo install expo-router expo-application expo-secure-store \
         expo-device @react-native-async-storage/async-storage \
         lucide-react-native

3. انسخ نفس فكرة lib/trial.tsx (نظام التجربة) إن كان تطبيقك الجديد
   رح يحتاج نظام تجربة/اشتراك مشابه — يمكنك أخذ نفس الملف من مشروع
   SmartPDF وتعديل السلوك حسب احتياج التطبيق الجديد.

4. انسخ نفس فكرة lib/i18n.tsx إن كان التطبيق يدعم لغتين.

5. صمّم لائحة الشاشات (Screens) والأدوات المطلوبة لتطبيقك الجديد بنفس
   أسلوب index.tsx (قائمة أدوات مقسّمة لمجموعات، كل أداة لها أيقونة
   ورابط شاشة خاص فيها).


القسم 6: تجهيز الخادم الخلفي (Backend) — إن احتجت واحداً
------------------------------------------------------------------------
نظام SmartPDF يستخدم:
   - FastAPI (Python) كخادم API.
   - Render.com لاستضافة الخادم مجاناً (Web Service).
   - Neon (neon.tech) كقاعدة بيانات Postgres مجانية ودائمة.
   - نظام تجربة (trial system) + لوحة تحكم إدارية (/admin) لمتابعة
     المستخدمين وإدارة تجاربهم.

إن كان تطبيقك الجديد يحتاج نفس الفكرة (تجربة مجانية + لوحة تحكم):
1. أنشئ حساب مجاني على https://render.com (يمكن الدخول بحساب GitHub).
2. أنشئ حساب مجاني على https://neon.tech لقاعدة بيانات Postgres.
3. اكتب خادم FastAPI جديد (main.py) بنفس فكرة خادم SmartPDF: نقطة
   /trial/check للتحقق من الجهاز، /trial/profile لحفظ بيانات المستخدم،
   ولوحة /admin لعرض وإدارة الأجهزة.
4. اربط رابط قاعدة بيانات Neon (Connection String) كمتغيّر بيئة
   DATABASE_URL في إعدادات الخدمة على Render.
5. انشر الخادم على Render، وضع رابطه في lib/config.ts بتطبيقك الجديد
   (SERVER_URL).


القسم 7: تجهيز المشروع لأول بناء (Build) مع EAS
------------------------------------------------------------------------
1. من داخل مجلد المشروع، اربطه بحساب Expo:
       eas init

   هذا الأمر بينشئ مشروع جديد بلوحة expo.dev ويضيف معرّف المشروع
   (projectId) تلقائياً لملف app.json.

2. جهّز ملف إعدادات البناء:
       eas build:configure

   هذا بينشئ ملف eas.json يحوي "بروفايلات" البناء الجاهزة:
   development (تطوير) / preview (معاينة) / production (نشر نهائي).

3. عدّل app.json بمعلومات تطبيقك الفعلية:
   - name: اسم التطبيق الظاهر للمستخدم
   - slug: اسم تقني فريد للمشروع
   - version: رقم الإصدار (مثال: 1.0.0)
   - android.package: معرّف فريد عالمياً، مثال:
         com.yourname.mynewapp
   - ios.bundleIdentifier: نفس الفكرة لآيفون، مثال:
         com.yourname.mynewapp
   - icon و splash: مسارات أيقونة التطبيق وشاشة البداية


القسم 8: أول بناء تجريبي (Development Build)
------------------------------------------------------------------------
مفيد لو تطبيقك يحتاج مكتبات فيها كود أصلي (Native) ما تدعمها Expo Go
العادية (متل expo-secure-store أحياناً، أو أي مكتبة SDK خارجية).

       npx expo install expo-dev-client
       eas build --platform android --profile development

بعد انتهاء البناء (تراقبه من رابط يظهر بالطرفية أو من لوحة expo.dev)،
ثبّت الملف الناتج (APK) مباشرة على جوال أندرويد للاختبار.


القسم 9: بناء نسخة النشر النهائية (Production Build)
------------------------------------------------------------------------
1. بناء أندرويد (ملف AAB جاهز لرفعه على Google Play):
       eas build --platform android --profile production

   عند أول مرة، EAS رح يسألك عن مفتاح التوقيع (Keystore) — اختر
   "Generate new keystore" وخلّي EAS يديره لك تلقائياً وبأمان.

2. بناء آيفون (يتطلب حساب Apple Developer المدفوع):
       eas build --platform ios --profile production

3. راقب تقدّم البناء من الرابط اللي يظهر بالطرفية، أو من لوحة
   expo.dev مباشرة (قسم Builds).


القسم 10: النشر على المتاجر
------------------------------------------------------------------------
Google Play:
1. أنشئ حساب Google Play Developer (رسوم 25$ لمرة واحدة):
       https://play.google.com/apps/publish/signup/
2. أنشئ تطبيق جديد من Google Play Console.
3. ارفع الملف مباشرة من الطرفية بدل الرفع اليدوي:
       eas submit --platform android

Apple App Store:
1. اشترك بعضوية Apple Developer Program (99$ سنوياً):
       https://developer.apple.com/programs/
2. أنشئ التطبيق من App Store Connect (اضغط "+" ثم "New App").
3. ارفع البناء مباشرة:
       eas submit --platform ios


القسم 11: التحديثات السريعة بدون إعادة نشر (EAS Update)
------------------------------------------------------------------------
لو عدّلت كود JavaScript فقط (بدون إضافة مكتبة جديدة فيها كود أصلي)،
ما في داعي تعيد كل عملية البناء والنشر بالمتاجر من الصفر:

       npx expo install expo-updates
       eas update --branch production --message "وصف التحديث"

هذا بيرسل التحديث مباشرة لأجهزة المستخدمين خلال دقائق، بدون المرور
بمراجعة المتجر — مفيد جداً لإصلاح الأخطاء الصغيرة وتعديلات النصوص.


خلاصة الأوامر الأساسية (مرجع سريع)
------------------------------------------------------------------------
npm install -g eas-cli                          تثبيت أداة EAS
eas login                                       تسجيل الدخول
npx create-expo-app MyNewApp                    إنشاء مشروع جديد
cd MyNewApp && npx expo start                   تشغيل التطوير المحلي
eas init                                        ربط المشروع بحسابك
eas build:configure                             تجهيز ملف eas.json
eas build --platform android --profile production   بناء نهائي أندرويد
eas build --platform ios --profile production        بناء نهائي آيفون
eas submit --platform android                   رفع لغوغل بلاي
eas submit --platform ios                       رفع لآبل ستور
eas update --branch production                  تحديث سريع بدون متجر


ملاحظات ختامية
------------------------------------------------------------------------
- كل الأوامر أعلاه تُنفَّذ من داخل مجلد مشروعك عبر الطرفية (Terminal
  أو Command Prompt أو PowerShell).
- لو احتجت مساعدة بأي خطوة أو واجهتك رسالة خطأ، ارجعلي بنص الرسالة
  وبساعدك أحددها بدقة.
- لو حابب أبني لك هيكلة المشروع الجديد فعلياً (الملفات والمجلدات
  الأساسية بنفس نمط SmartPDF)، قلّي تفاصيل التطبيق الجديد (اسمه،
  وظيفته الأساسية) وبجهزلك الملفات جاهزة للتنزيل مباشرة.
